# 🛡️ Politique de Sécurité | LANAZRION_OS

## Vision de Sécurité
La protection des actifs numériques et de l'intégrité des données territoriales (Matrice des 334 Terrils) est une priorité absolue du protocole **[SYMBIOSIS_ALPHA]**. Nous appliquons des standards de précision industrielle pour garantir la souveraineté de l'Hémisphère Droit.

## Versions Supportées

Nous assurons la maintenance et les correctifs de sécurité uniquement sur les cycles de déploiement actifs.

| Version | Statut | Notes |
| :--- | :--- | :--- |
| **v1.1.x (Actuelle)** | ✅ Supporté | Version stable connectée à l'API Gemini 1.5. |
| **v1.0.x** | ❌ Obsolète | Première release alpha (Migration requise). |
| **Bêta / Dev** | ⚠️ Limité | Versions de test pour le mode Patrimoine (Terrilium). |

## Signalement d'une Vulnérabilité

Si vous identifiez une faille de sécurité (exposition de clé API, fuite de données de la matrice, ou bug système), suivez ce protocole :

1. **Ne pas ouvrir de "Issue" publique :** Pour éviter l'exploitation de la faille par des tiers.
2. **Contact Direct :** Envoyez un rapport détaillé à l'adresse stratégique : `security@lanazrion.com` (ou via ton profil LinkedIn de référence).
3. **Contenu du Rapport :** - Description de la vulnérabilité.
   - Étapes pour reproduire l'incident.
   - Impact potentiel sur le système **Kaleidoland**.

### Délais et Engagement
- **Accusé de réception :** Sous 24 heures (Fuseau horaire Europe/Paris).
- **Évaluation :** Une analyse technique sera effectuée sous 72 heures.
- **Résolution :** Selon la criticité, un correctif sera déployé en priorité absolue via un "Hotfix" sur la branche `main`.
